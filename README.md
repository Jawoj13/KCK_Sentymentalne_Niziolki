# KCK_Sentymentalne_Niziolki

## Do rozważenia:
* Architektura: Sprzęt z wejściami, program lokalny
* Baza danych do historii treningów/ustawień
* Podsumowanie treningu, jaka biblioteka?
* GUI: Python PyGame/PyQt/TK-Inter
* Interface głosowy: Vosk/pyttsx3/SpeechRecognition/PyAudio
* Przetwarzanie obrazu wyłącznie przez komputer
* Nie będzie planowania i powiadomień, wymagałoby to ciągłego działania w tle albo implementacji serwera
* Kontrola onclick czy głosowa, problemem odległość wymagana do zmieszczenia się z bronią w kadrze (GESTY!!!?) - potrzebne tylko po uruchomieniu ćwiczenia

## Problemy:
* Dynamika ćwiczenia może sprawiać problem z wyłapywanie konkretnych klatek ruchu
* Miecz jako 'przedłużenie ręki'
* Kontrola braku ruchu (zamach jest błędem)
* Oznaczenie końca broni tak aby było łatwe do wykrycia (Żółta taśma izolacyjna na końcu broni)
